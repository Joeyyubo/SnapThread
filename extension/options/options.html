SnapThread options

Where it works: You can use SnapThread on any website loaded over http:// or https://—production, staging, localhost, docs, or internal tools—not only dev previews. Browser-only URLs such as chrome://, the Chrome Web Store, and most file:// pages cannot run the capture script.

PR comments use a classic GitHub Personal Access Token. For least access, enable public_repo for public repos, or repo for private ones. The token is stored only in this extension's local storage on your device.

Screenshots: The extension needs access to http://* and https://* pages. If you see "Either the '<all_urls>' or 'activeTab' permission is required", reload this extension on chrome://extensions, then refresh the tab you are reviewing. That error usually means the page URL was not covered by an older, narrower permission list (e.g. http://192.168…).

Post to GitHub with images: Comments use the same API for PRs and issues. PNGs go under .snapthread/pr-<n>/ on the PR head branch, or .snapthread/issue-<n>/ on the repo default branch for plain issues. Your token needs repo (classic) or fine-grained Contents: Read and write. Fork PRs require push access to the fork that owns the head branch.
